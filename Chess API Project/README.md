# Chess API Project ♟

As an avid chess player, I’m always eager to explore new strategies and refine my skills. One day, while playing on the Chess.com app, I began to wonder if I could apply statistical analysis to my gameplay history. A quick Google search revealed that Chess.com offers a public API, and in that moment, inspiration struck. I decided to channel my curiosity into a project, diving into the data to (hopefully) uncover meaningful insights and patterns.

The first program I made was a Search Tool to collect and display game data for a single user. I made this tool for two reasons, to see if there were any high-level patterns in my gameplay history, and to get used to working with the API. One insight I discovered was that I generally had a higher level of success when I played the sicilian defense opening, however for one particular variation of that opening I lost almost all my games. I now know to keep an eye out for that variation from now on.

I am currently working on a program to analyze the moves that were played in each game, as well as the games of players from a similar skill level as myself. I hope to find out whether certain opening moves lead to higher chances of winning, as well as the likelihood of making a "blunder" under certain conditions.
